# 代码分析与方案设计

## 使用方法

1. **读取代码文件**：使用 read 工具读取用户指定的文件路径
2. **分析代码逻辑**：理解文件实现的功能、关键类/函数、依赖关系
3. **输出方案设计**：按以下结构组织内容
   - 功能概述
   - 整体架构思路
   - 关键点详细说明（核心逻辑、数据流、边界处理等）
4. **保存结果**：生成Markdown格式文档

## 输出格式模板

```markdown
# [文件名] 方案设计

## 一、功能概述
[简述这个文件/模块实现的核心功能]

## 二、整体思路
[从整体角度描述实现方案]

## 三、关键点详细说明

### 3.1 [关键点1]
[详细说明]

### 3.2 [关键点2]
[详细说明]

...

## 四、数据结构（如适用）
[相关接口、数据结构定义]
```

## 保存位置

- 输出目录：`~/Desktop/前端需求方案设计/`
- 如果目录不存在，先创建
- 文件命名：`[原文件名]-方案设计.md`
